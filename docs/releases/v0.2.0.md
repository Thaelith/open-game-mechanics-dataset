# Open Game Mechanics Dataset v0.2.0

Release date: 2026-06-30

## Summary

v0.2.0 turns the project from a structured mechanic catalog into a graph-aware dataset foundation. The release keeps the dataset engine-agnostic and schema-valid while adding optional typed relationships, rough scope metadata, graph validation, report artifacts, and the first static Mechanic Mixer MVP.

This is still not a complete database of every game mechanic. It is a practical v0.2 foundation for browsing, contribution review, AI-agent planning prompts, and early concept-scope analysis.

## What Changed

- 223 mechanic JSON files remain valid against the schema.
- `dataset.json` now reports `schema_version: "0.2.0"`.
- Optional `relationships` entries describe typed graph edges such as `requires`, `supports`, `enhances`, `conflicts_with`, `soft_conflicts_with`, `feeds`, `consumes`, and `unlocks`.
- Optional `scope_profile` entries describe rough MVP role, implementation cost, design cost, tuning cost, content cost, networking risk, save/load risk, and UI risk.
- 175 mechanics now include typed relationships and scope profiles.
- The dataset contains 526 typed relationships.
- Graph validation and graph coverage reporting are available in `tools/`.
- The static browser includes a Mechanic Mixer MVP.
- The detail panel shows typed relationships and scope profile data when available.
- The Mixer has deterministic unit tests and 10 scenario QA fixtures.

## Why It Matters

Flat browse links are useful for discovery, but future planning tools need stronger structure. Typed relationships make it possible to identify missing dependencies, conflict risks, support mechanics, and scope pressure without inventing unsupported claims.

The Mechanic Mixer uses that structure as a first client-side prototype. It does not generate a full game idea or call an AI backend; it explains what the selected mechanics imply according to the dataset.

## Known Limitations

- Relationship coverage is broad but not complete.
- Scope scores are rough relative planning signals, not production estimates.
- The Mixer does not replace design review, technical review, accessibility review, or playtesting.
- Incoming graph suggestions are lightweight.
- There is no full visual graph browser yet.
- There is no AI backend or generated game-concept service.
- Example game titles are short factual references, not claims about proprietary implementation.
