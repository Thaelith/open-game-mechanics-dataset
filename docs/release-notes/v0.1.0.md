# Open Game Mechanics Dataset v0.1.0

Release date: 2026-06-29

## Status

v0.1.0 is an early MVP release. It is intended to be useful immediately while making the limits of the dataset clear: the project is an expandable open dataset, not a claim to contain every mechanic or every design pattern used in games.

## Included

- 223 schema-valid mechanic JSON files.
- A strict mechanic schema at `schema/mechanic.schema.json`.
- A generated `dataset.json` index for search, filtering, and downstream tooling.
- Documentation for contributors, taxonomy, difficulty scales, mechanic writing, and AI-agent usage.
- GitHub issue templates, pull request template, and validation workflow.

## Tooling

The release includes Python 3.11+ tools for validation and dataset maintenance:

- `tools/validate_dataset.py` validates mechanic files, duplicate IDs, and mechanic references.
- `tools/generate_index.py` builds or checks `dataset.json`.
- `tools/check_links.py` checks repository-local documentation links.
- `tools/generic_lint.py` reports advisory quality warnings without failing CI.
- `tools/category_coverage_report.py` reports dataset coverage by category, difficulty, dimension, genre, status, and source confidence.
- `tools/search_dataset.py` provides basic command-line search by ID, category, genre, tag, and text query.

## Quality Notes

The v0.1 dataset has received targeted quality passes for high-impact movement, combat, camera, traversal, UI/UX, procedural generation, and related entries. Implementation notes are written to stay engine-agnostic while still naming useful state, event sources, validation rules, and engine-specific primitives where appropriate.

Remaining work should prioritize deeper review over raw volume.
