# Category Guide

Choose the category by primary player-facing purpose.

- If it changes avatar movement, use `movement`.
- If it is a precision platform challenge or jump/wall interaction, use `platforming`.
- If it moves the player through world structure using tools or anchors, use `traversal`.
- If it creates attack, defense, hit reaction, or targeting rules, use `combat`.
- If it frames the world, use `camera`.
- If it is mostly an interface behavior, use `ui_ux`.
- If it adapts controls, perception, timing, or difficulty for players, use `accessibility`.

Cross-category mechanics should use one primary category and link related mechanics rather than duplicating entries.
